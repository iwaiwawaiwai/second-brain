週次レビューを実行してください。

## 実行フロー

### 1. Inbox処理
`00_Inbox/` の全アイテムを確認する。
各アイテムについて以下を判断する。

- 来週の課題として取り組むもの → `30_Areas/課題リスト.md` のバックログに追加
- 知識・参考資料として残すもの → `40_Resources/` に移動
- 完了・不要なもの → 削除または `99_Archives/` に移動

### 2. 課題リストの整理
`30_Areas/課題リスト.md` を開く。
今週のフォーカスの完了状況を確認し、以下を更新する。

- 完了した課題 → 完了済みセクションに移動
- 未完了で来週も続ける課題 → そのまま残す
- 優先順位の見直し → バックログの並び順を調整する
- 来週のフォーカス（1〜2個）を選んで「今週のフォーカス」欄を更新する

### 3. 放置PJ検出
`20_Projects/` で1週間以上更新のないプロジェクトを検出する。

### 4. 進捗サマリー
今週完了したタスク・課題の一覧を作成する。

### 5. 学びの抽出
`10_Journal/` から今週の学びを要約する。

### 6. メンタルパターンの振り返り
今週のコーチング対話録・voice-note・モヤモヤメモから感情の波を読み取る。

- 何が不安・焦り・混乱のトリガーだったか
- 何で落ち着けたか・解決できたか
- 繰り返しているパターンはあるか

### 7. 来週の処方箋
タスク優先順位と来週の自分へのメンタル準備（一言）を提案する。

## 出力
レビュー結果を `brain/10_Journal/` に保存する。
フォーマット: `YYYY-MM-DD_weekly-review.md`

## 出力テンプレート

```
## 今週の完了
- {完了した課題}

## 来週のフォーカス
- {課題1}
- {課題2}

## 今週の感情パターン
{日付ごとに: 状態・トリガー・解決したことを箇条書き}

## 繰り返しているパターン
- {気づき}

## 来週の処方箋
{来週の自分への一言。役割・帽子・心構えなど具体的に}
```
