作業セッションの開始・中断・再開・確認を行う。

## 呼ばれたときの動作

1. `brain/00_Inbox/作業中.md` を読む
2. `brain/30_Areas/課題リスト.md` を読む
3. 取り組む課題を以下の優先順で1つ選んで提示する：
   - 作業中.md に中断中の課題があればそれを最優先
   - なければ今週のフォーカスの未完了を上から1つ
4. 以下を表示する：

```
## 今やる課題
{選んだ課題名}

## 進捗
{作業中.mdの「ここまで進んだこと」/ 初回なら「未着手」}

## 次にやること
{作業中.mdの「再開したらまずやること」/ 初回なら最初のステップを提示}
```

5. ユーザーに確認する：
   - 「これでいく」→ そのまま作業開始
   - 「別のにする」→ 今週のフォーカス一覧を表示して選んでもらう
   - 「続けます」→ そのまま作業再開
   - 「中断します」「タイマー終了」「ポモドーロ終了」→ 以下を記録する：
     - やったこと
     - 次にやること
     - 気になったこと（課題リストに追加するか確認）
   ※ 作業が完了していなくても同じ手順。未完了のまま中断してよい
   - 「完了しました」→ 以下を実行する：
     - 課題リスト.md の該当課題を完了済みに移動する
     - 作業中.md をクリアする
     - 「次の課題に移りますか？」と確認する
     - 移る場合 → 課題リストのバックログから次を提示する
     - 移らない場合 → そのまま終了（中断と同じ扱い）

## 作業中.mdのフォーマット

```
## 最終更新: {HH:MM} {開始 or 中断}

### 取り組んでいる課題
{課題名}

### ここまで進んだこと
{箇条書き}

### 再開したらまずやること
{1行}

### 気になったこと（Inbox行き）
{箇条書き、なければ空}
```

## ルール

- 課題の完了・追加は即座に課題リスト.mdに反映する
- 気になったことは課題リストへの追加判断をその場で行う
- 作業中.mdは常に最新の1セッション分だけ保持する（上書き）
