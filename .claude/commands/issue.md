自然言語からGitHub Issueを作成してください。

入力: $ARGUMENTS

## 実行フロー
1. 入力テキストを解析する
2. 以下を自動判定する:
   - タイトル（簡潔に）
   - 本文（背景・目的・完了条件）
   - ラベル（bug, feature, enhancement, etc.）
   - マイルストーン（該当するものがあれば）
   - 優先度（P0-P3）
3. GitHub Issue を作成する
4. 関連するbrain内ノートがあればIssue本文にリンクする

## ルール
- タイトルは動詞で始める
- 完了条件を必ず含める
- 曖昧な場合は確認してからIssueを作成する
