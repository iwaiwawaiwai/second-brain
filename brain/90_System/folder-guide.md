# Brain フォルダ運用ガイド

## フォルダ構成

```
brain/
├── 00_Inbox/      # 受信トレイ
├── 10_Journal/    # 日記・振り返り
├── 20_Projects/   # プロジェクト
├── 30_Areas/      # 継続エリア
├── 40_Resources/  # 知識・参考資料
├── 90_System/     # 運用ルール（このファイル）
└── 99_Archives/   # 完了・保管
```

---

## 各フォルダの使い方

### 00_Inbox — 受信トレイ
- **何でも最初はここに入れる**
- メモ、音声ノート、アイデア、会議メモなど
- 週1回見直して他のフォルダに移動する
- ここに長く置きすぎない

### 10_Journal — 日記・振り返り
- 日付ベースの記録
- 日次・週次の振り返り
- 感情や状態の記録
- ファイル名例：`2026-03-23_weekly-review.md`

### 20_Projects — プロジェクト
- **終わりがあるもの**を置く
- 例：kaorilogo、hiroshima-ai-sandbox、退職・引き継ぎ
- 完了したら `99_Archives/` に移動する
- プロジェクトごとにサブフォルダを作る

### 30_Areas — 継続エリア
- **終わりがなく、ずっと続くもの**を置く
- 例：健康管理、学習、家計、習慣
- Projectsとの違い：ゴールではなく「基準を保つもの」

### 40_Resources — 知識・参考資料
- 特定のプロジェクトに依存しない**永続的な知識**
- 例：技術メモ、ビジネス知識、業界情報
- 後から何度も参照するもの

### 90_System — 運用ルール
- このガイドのような**brainの使い方**に関するもの
- テンプレート
- 運用を変えたときはここを更新する

### 99_Archives — 完了・保管
- 完了したProjectsを移動する
- 削除はしないが、普段は見ない

---

## ワークフロー

```
入力 → 00_Inbox
         ↓（週次レビュー）
    ┌────────────────┐
    │ 20_Projects    │ ← 進行中のプロジェクト
    │ 30_Areas       │ ← 継続的な管理
    │ 40_Resources   │ ← 知識として残す
    │ 10_Journal     │ ← 日記・振り返り
    └────────────────┘
         ↓（完了）
    99_Archives
```

---

## 判断に迷ったとき

| 迷い | 答え |
|------|------|
| ProjectsかAreasか | 「完了する日がある」→ Projects |
| InboxかProjectsか | まず Inbox、整理するときに移動 |
| 削除かArchivesか | 迷ったら Archives に入れる |
