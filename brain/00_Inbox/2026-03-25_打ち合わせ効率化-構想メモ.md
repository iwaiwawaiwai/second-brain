## Claude Code コマンドについての理解

- `/voice-note` などのコマンドは「指示書」に過ぎない。会話は終わらない
- コマンドは会話の途中でも何度でも呼び出せる
- **構想メモに追記してもらう形が、ファイルを育てるうえで適切**
  - 例：「構想メモにこの気づきを追記して」と依頼する

## リアルタイム議事録の使い方（新しい気づき）

ホワイトボードの代わりにClaude Codeをプロジェクターに映して、
ヒアリング中にリアルタイムでメモを構造化していく使い方。

```
ホワイトボード              Claude Code（プロジェクター）
書き殴り → 消える           話しながら構造化 → そのまま保存
整理は後でやり直し           会議終了 = 議事録完成
```

**流れ：**
1. ヒアリング中に「今言ったこと追記して」と話しかける
2. 画面上でリアルタイムに整理されていく
3. 相手も見ながら認識のズレをその場で修正できる
4. 会議が終わったらそのままbrainに保存

→ **会議の進め方自体が変わる可能性がある**

### 入力の合わせ技（現実的な運用案）

**案1：ターン切り替え方式**
- 相手が話している間 → キーボードでキーワード・断片メモ
- 自分のターンになった時 → 音声入力で補足・整理を依頼
- キーボード（速い・静か）と音声（まとまった文章）の自然な分業

**案2：Aqua Voice 常時ON方式（現実的な最適解かも）**
- Aqua Voice を常時ONにして周囲音ごと録音・文字化
- 相手の発話も自分の発話もそのまま拾う
- 一区切りついたタイミングで「ここまでをまとめて」とClaudeに投げる
- キーボード入力不要・聞き漏らしなし・区切りで自動構造化
